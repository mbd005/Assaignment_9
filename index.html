1. Execution Context

What is Execution Context in JavaScript?

Everything inside javaScript happends inside an execution context. when the first time script run execution context made automatically. This has two phase , first is memory creation and second is code execution.

Explain Global Execution Context and Function Execution Context.

when first script load by default Global execution context create by browser. in global execution context all the variable and function main code always stored. It's stay at the bottom of the call stack.
-- Function Execution Context --
in FEC when something invoked a function the function execution context run the function and execute it. it's also has two phase. in memory creation holds the function’s parameters, local variables. And in second phase the code execute line by line. If it's return something then it goes previous place where it was invoked.

What happens during the Creation Phase and Execution Phase?

Inside creation phase all the variable and function stored in a memory. at the start the default value of variable is (undefined). In Execution phase code run by line by line. It's a synchronus language.

5. Promise

What is a Promise?

Promise is a object that returns the result of a work. It may be fullfieled or rejected.

Explain Promise States:

Pending

Pending means that when code inter in promise states it's wait there to receive data from another source.

Fulfilled

Fulfilled means in certain condition or rules have matched as user requests, so the word of promised fullfieled or resolved.

Rejected

rejected means the condition was not matched with user requests so the result have been reject or the data was not found.

6. Async/Await

What is async?

In JavaScript, async is a keyword used to declare an asynchronous function that always returns a Promise.

What is await?

await is an operator used to pause the execution of an async function until a Promise settles (resolves or rejects).

How does async/await improve code readability?

It makes asynchronous code look like synchronous. Also it's get easy to read for other because it can control using variable.And try-catch make it easy to find error.
